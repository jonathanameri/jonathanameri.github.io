Front-End Tech

For this application, I recommend using the React and React Native frameworks for the front-end. React Native is ideal for this project because the client is looking for an application that works on both iOS and Android devices, as well as the web. With React Native, our development team can write code once and publish the application for multiple platforms. In addition, React is the most popular JavaScript framework, so there is an abundance of documentation and additional libraries available online. I also suggest using the Expo library, which provides us with access to pre-made React components for features such as using the device's camera, local storage, and geolocation data. Since the client is looking to store images and other text information about each POI, as well as the location of the POI, these features are necessary. To handle offline usage for the researchers, I recommend using the local storage of a researchers device to store data about a POI, and have the app make interval connection requests to the backend to eventually permanently store the data. This will ensure that in the case that a researcher loses internet connection, the data they collect while offline will make it to our backend.

For mapping, I suggest that the DXG uses Mapbox for this project. They offer a JavaScript library called 'Mapbox GL JS' which allows for integration of maps into React apps. This would be the easiest and fastest way to develop a map with points of interest directly on it. Mapbox is a well known service, so it has a ton of documentation on their site which would aid the development process. Additionally, since the DXG has already used Mapbox, our development team would have no issue with using it for this project.
